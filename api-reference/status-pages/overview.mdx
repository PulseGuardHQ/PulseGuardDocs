---
title: 'Status Pages API Overview'  
description: 'Create and manage public status pages for your services.'
---

The Status Pages API allows you to create and manage public status pages that display the real-time status of your monitored services.

## Features

- **Public Status Pages** - Branded public pages for your services
- **Custom Domains** - Use your own domain for status pages
- **Incident Management** - Create and manage service incidents
- **Maintenance Windows** - Schedule and communicate maintenance
- **Subscriber Notifications** - Email/SMS notifications for status changes
- **RSS/Calendar Feeds** - Machine-readable status feeds
- **Widget Embedding** - Embed status widgets on your website

## Plan Limits

Status page limits by plan:

- **Free Plan** - 1 status page
- **Standard Plan** - 1 status page  
- **Pro Plan** - 2 status pages
- **Enterprise Plan** - 3+ status pages

## Authentication

### Private Endpoints (Management)
Require Sanctum authentication and plan verification:

```bash
Authorization: Bearer {your_token}
```

### Public Endpoints (Viewing)
No authentication required - publicly accessible:

```bash
# Public status page
GET /status/{slug}

# Public API data
GET /api/status/{slug}
```

## Status Page Structure

Each status page contains:

- **Services** - List of monitored services/components
- **Current Status** - Real-time operational status
- **Incident History** - Past incidents and resolutions
- **Maintenance Schedule** - Upcoming maintenance windows
- **Metrics** - Optional performance charts

## Service Status Types

- `operational` - Service is working normally
- `degraded_performance` - Service is slower than usual
- `partial_outage` - Some features unavailable
- `major_outage` - Service is completely down
- `maintenance` - Planned maintenance in progress

## Incident Types

- `investigating` - Issue identified, investigating
- `identified` - Root cause found
- `monitoring` - Fix deployed, monitoring
- `resolved` - Issue completely resolved

## Public API Format

Public status pages return data in this format:

```json
{
  "page": {
    "id": "acme-corp",
    "name": "Acme Corp Status",
    "url": "https://status.acme.com",
    "updated_at": "2024-01-15T10:30:00Z"
  },
  "components": [
    {
      "id": "web-app",
      "name": "Web Application", 
      "status": "operational",
      "description": "Main web application"
    }
  ],
  "incidents": [
    {
      "id": "inc-123",
      "name": "Database Performance Issues",
      "status": "resolved",
      "created_at": "2024-01-15T08:00:00Z",
      "updates": [
        // Incident timeline
      ]
    }
  ]
}
```

## Customization Options

- **Branding** - Custom logos, colors, and styling
- **Custom Domain** - Use your own domain (status.yourcompany.com)
- **Contact Information** - Support links and contact details
- **Social Media** - Links to your social accounts
- **Custom CSS** - Advanced styling options

## Notification Subscriptions

Users can subscribe to status updates via:
- **Email** - Incident and maintenance notifications
- **SMS** - Critical alerts (Enterprise only)
- **Webhooks** - Automated system notifications
- **RSS** - RSS feed subscriptions

## Embedding

Status pages can be embedded using:

### Status Widget
```html
<iframe src="https://status.yourcompany.com/widget" 
        width="300" height="200"></iframe>
```

### Status Badge
```html
<img src="https://status.yourcompany.com/badge.svg" alt="Status">
```
