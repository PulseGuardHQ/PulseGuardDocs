The Statistics API provides comprehensive analytics and historical data for your monitored resources.

## Available Statistics

### Domain Statistics
- **Response Time Analytics** - Historical response time data with statistical analysis
- **Availability Metrics** - Uptime/downtime statistics and trends
- **Performance Trends** - Long-term performance analysis

### Device Statistics  
- **Resource Usage** - CPU, memory, disk usage over time
- **Network Metrics** - Bandwidth and connectivity statistics
- **System Performance** - Boot times, process counts, system load

### Service Statistics
- **Service Availability** - Uptime statistics for monitored services
- **Response Times** - Service-specific performance metrics
- **Error Rates** - Failure and timeout statistics

## Authentication

Statistics endpoints require:
- **Sanctum Authentication** - Valid API token
- **Ability Permissions** - `api:stats,read` for read access

## Rate Limiting

Statistics API follows standard rate limits:
- **60 requests per minute** for authenticated users
- **Caching** - Results cached for 5 minutes for performance

## Response Format

All statistics endpoints return data in this format:

```json
{
  "success": true,
  "message": "Statistics retrieved successfully",
  "data": {
    "period": "24h",
    "metrics": {
      // Statistical data
    },
    "summary": {
      "average": 123.45,
      "minimum": 89.12,
      "maximum": 234.67,
      "median": 115.23
    }
  }
}
```

## Time Periods

Most statistics endpoints support these time periods:
- `1h` - Last hour
- `24h` - Last 24 hours (default)
- `7d` - Last 7 days
- `30d` - Last 30 days
- `all` - All available data

## Data Sampling

For performance, large datasets are automatically sampled:
- **< 1000 points** - All data returned
- **> 1000 points** - Intelligent sampling applied
- **Time intervals** - Adjusted based on requested period
