## Setting Up Slack Integration

Connect PulseGuard to Slack to receive real-time notifications about your monitored resources:

<Steps>
  <Step title="Navigate to Integrations">
    Go to Settings > Integrations in your PulseGuard dashboard
  </Step>
  <Step title="Select Slack">
    Click on the Slack card in the available integrations
  </Step>
  <Step title="Authorize PulseGuard">
    Click the "Connect to Slack" button and authorize PulseGuard to access your Slack workspace
  </Step>
  <Step title="Select Channels">
    Choose which Slack channels should receive notifications
  </Step>
  <Step title="Configure Notifications">
    Select which events should trigger notifications (e.g., domain down, SSL expiry warnings)
  </Step>
  <Step title="Save Configuration">
    Click "Save" to activate the integration
  </Step>
</Steps>

## Using Webhooks

Webhooks allow you to integrate PulseGuard with custom services and applications:

<AccordionGroup>
  <Accordion title="Webhook Setup">
    1. Go to Settings > Integrations > Webhooks
    2. Click "Add Webhook"
    3. Enter the webhook URL that will receive the data
    4. Select which events should trigger the webhook
    5. Choose the data format (JSON or Form Data)
    6. Save the webhook configuration
  </Accordion>
  
  <Accordion title="Webhook Payload Format">
    ```json
    {
      "event": "domain_down",
      "resource": {
        "type": "domain",
        "id": "d-12345",
        "name": "example.com",
        "url": "https://example.com"
      },
      "status": {
        "code": "down",
        "status_code": 503,
        "message": "Service Unavailable"
      },
      "timestamp": "2023-07-15T14:22:31Z",
      "additional_data": {
        "response_time": null,
        "previous_status": "up",
        "down_since": "2023-07-15T14:22:31Z"
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Webhook Events">
    - `domain_down`: Triggered when a domain becomes unavailable
    - `domain_up`: Triggered when a domain recovers
    - `ssl_expiry_warning`: Triggered when an SSL certificate is nearing expiration
    - `device_alert`: Triggered when a device metric exceeds thresholds
    - `anomaly_detected`: Triggered when an anomaly is detected by AI
    - `report_generated`: Triggered when a report is generated
  </Accordion>
  
  <Accordion title="Testing Webhooks">
    You can test your webhook integration by clicking the "Test" button in the webhook configuration page. This will send a test payload to your webhook URL to verify that it's correctly configured.
  </Accordion>
</AccordionGroup>

## SMS Notifications (Pro & Expert Plans)

Set up SMS notifications for critical alerts:

<Steps>
  <Step title="Navigate to Integrations">
    Go to Settings > Integrations in your PulseGuard dashboard
  </Step>
  <Step title="Select SMS">
    Click on the SMS card in the available integrations
  </Step>
  <Step title="Add Phone Number">
    Enter the phone number that should receive SMS alerts
  </Step>
  <Step title="Verify Number">
    Enter the verification code sent to your phone
  </Step>
  <Step title="Configure Alert Types">
    Select which critical alerts should be sent via SMS
  </Step>
  <Step title="Save Configuration">
    Click "Save" to activate SMS notifications
  </Step>
</Steps>

<Note>
  SMS notifications are only available on Pro and Expert plans and are primarily intended for critical alerts due to SMS delivery costs.
</Note>

## Custom Integrations via API

For advanced integration needs, you can use our [API](/api-reference/introduction) to build custom integrations:

```bash
# Example API call to get domain status
curl -X GET "https://api.pulseguard.app/v2/domains/{domain_id}/status" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

<Note>
  API access is available on Pro and Expert plans. Visit our [API Documentation](/api-reference/introduction) for more details.
</Note>
